参赛组别：研究生组
版本号码：V1.0

============================================================================
一、项目概况
============================================================================

1. 背景和基础

云博物馆（Cloud Museum）项目源于团队对陕西丰富历史文化遗产数字化传承的深入
思考。陕西省作为中华文明的重要发源地，拥有周、秦、汉、唐等十三朝古都遗址，
馆藏文物数量庞大、种类繁多。然而，传统博物馆在参观体验上存在时空受限、讲解
资源有限、互动性不足等痛点，大量珍贵文物"养在深闺人未识"。

项目灵感来自两方面：一是国家"十四五"文物保护和科技创新规划中明确提出"推动
博物馆数字化、智能化发展"的战略方向；二是2024年陕西省"数字陕西"建设方案
强调运用人工智能技术赋能文旅产业升级。团队以此为切入点，设计并开发了基于AI
大模型的智慧云博物馆平台。

团队构成方面，本项目由西安工程大学研究生团队主导开发，成员涵盖计算机科学与
技术、人工智能、视觉交互设计等专业方向。团队成员在前端开发（Vue 3 + Vant 4 /
Element Plus）、后端架构（Spring Boot + JPA）、AI应用（LangChain4j + 大模型
集成）和数据库设计（MySQL + Redis）方面具备完整的工程能力。

已有工作基础包括：完成了完整的后端RESTful API设计与实现，涵盖藏品管理、分类
管理、预约管理、用户认证等核心业务模块；实现了基于Vue 3的用户端H5应用和管理
端Web应用；集成了基于DeepSeek大模型的AI智能讲解功能，实现了流式SSE对话和
藏品图文讲解自动生成；实现了基于Google Model Viewer的3D模型展示功能，支持
Web端实时交互浏览和AR预览；数据库层面设计了完善的藏品信息模型，支持中文全文
检索。


2. 场景和价值

本项目主要面向以下应用场景：

（1）线上数字博物馆参观场景：用户通过手机H5应用即可随时随地浏览博物馆藏品，
查看高清图片、3D模型，获取详细的文物信息，突破传统博物馆的时空限制，实现
"把博物馆装进口袋"。

（2）AI智能导览讲解场景：在藏品详情页中嵌入AI智能讲解助手，用户可输入自然
语言问题，AI基于藏品档案数据进行专业、亲切的讲解回答。同时支持AI自动生成
结构化图文讲解内容（包含历史背景、关键知识点、趣味冷知识、文化价值等），让
每一位用户都能获得"私人讲解员"级别的参观体验。

（3）文博教育科普场景：高校历史文化课程、中小学研学活动可通过平台进行在线
文物教学，AI讲解功能支持语音播报，适合不同年龄段用户获取历史文化知识，助力
文化自信建设。

（4）博物馆数字化管理场景：管理端提供完整的藏品信息管理、预约审核、数据统计
等功能，帮助博物馆实现从纸质档案到数字化管理的转型升级，大幅提升运营效率。

社会价值方面：本项目通过AI技术降低了历史文化传播的门槛，使优质文物资源能够
服务更广泛的公众群体。在数字经济与实体经济深度融合的背景下，云博物馆为陕西
乃至全国文博产业的数字化转型提供了可复制的技术范式，对文化遗产保护、文化
传承创新、文旅产业发展具有积极的推动作用。

市场调研方面：现有竞品（如故宫博物院App、云游敦煌等）在AI交互深度上仍有不足，
大多停留在预设语音讲解阶段，缺乏真正的多轮对话智能问答能力。本项目的AI智能
讲解基于大语言模型，能够根据用户个性化问题进行针对性解答，是传统语音导览的
升级替代方案。


3. 所需支持

项目实施过程中需要以下支持：

（1）算力支持：AI大模型推理需要GPU算力支持，当前项目使用DeepSeek API云端
调用，推理在服务端完成。后续如部署华为盘古大模型或在本地进行模型微调，需要
配备NVIDIA GPU服务器（如A100/昇腾910）。当前阶段可通过华为云ModelArts平台
获取弹性算力资源。

（2）数据资源：需要与博物馆合作获取高清文物影像数据、3D扫描模型数据以及权威
的文物档案资料。当前阶段团队通过公开的文物数据库和自行搜集的标准化数据完成
了原型开发，正式上线需要与文物主管部门建立数据合作。

（3）云服务资源：项目当前部署于本地开发环境，正式发布需要云服务器（推荐华为
云ECS）、对象存储（华为云OBS用于图片和3D模型文件存储）、云数据库（华为云
GaussDB或RDS）等基础设施。

（4）技术培训：团队对华为云服务生态（ModelArts AI平台、OBS对象存储、ECS云
服务器等）的使用需要进一步培训，以更好融入华为技术体系，体现竞赛技术深度。

（5）硬件设备：3D扫描仪（用于文物高精度三维建模）、高清摄影设备（用于文物
影像采集）等硬件资源，有助于丰富平台内容。

二、项目规划
============================================================================

1. 整体目标

参赛期间（2026年5月—2026年8月），本项目的整体目标如下：

（1）完成可展示的原型系统：提供完整的用户端H5和管理端Web系统，支持藏品浏览、
搜索、3D展示、AI智能讲解、在线预约等核心功能，形成"看—查—问—约"的完整
参观体验闭环。

（2）AI功能深度验证：实现并验证基于大模型的AI智能讲解功能在实际场景中的有效
性，包括多轮对话交互、自动图文讲解生成、语音播报等，形成至少一项AI技术在
文博领域的应用案例。

（3）华为云技术融合验证：完成与华为云基础服务的集成（云服务器部署、对象存储、
ModelArts AI平台等），验证云上架构的可行性和性能指标。

（4）行业初步验证：收集至少50名真实用户的试用反馈，评估AI讲解的准确率、用户
满意度和系统性能指标，形成用户体验报告。

（5）竞赛成果输出：完成项目文档撰写、演示视频制作和系统演示环境搭建，为参加
第八届中国研究生人工智能创新大赛做好充分准备。


2. 技术创新点

本项目的技术创新点主要体现在以下几个方面：

（1）基于RAG架构的AI智能讲解系统：区别于传统博物馆固定语音导览或预设问答
机器人，本项目基于LangChain4j框架构建了检索增强生成（RAG）式AI讲解系统。
系统将藏品档案信息（名称、朝代、材质、描述、文化意义等结构化+非结构化数据）
整合为系统提示词，结合DeepSeek大模型的自然语言生成能力，实现面向具体藏品的
多轮对话讲解。关键技术细节包括：通过Redis实现对话历史的持久化和24小时过期
管理，确保用户数据隐私；基于SSE（Server-Sent Events）协议实现AI回复的流式
逐字推送，优化用户等待体验；采用独立的线程池处理AI请求，避免阻塞Tomcat工作
线程。

（2）多模态展示与交互技术：平台集成了Google Model Viewer Web Component，
支持在移动端和PC端直接渲染.glb格式的3D文物模型，用户可通过触摸或鼠标拖拽、
缩放实现全角度观察。同时结合语音合成（Web Speech API）实现AI讲解内容的语音
播报，提供"看—听—问"的多模态交互体验。

（3）全栈前后端分离架构：采用Spring Boot 3.2 + Vue 3的全栈分离架构，基于
JWT Token实现统一认证授权，同时支持用户端和管理端两套前端应用。后端通过
RESTful API统一提供服务，使用Spring Data JPA实现ORM映射，MySQL全文索引支持
中文藏品检索，Redis缓存加速热点数据访问。

（4）智能藏品推荐机制（规划中）：基于用户行为数据的智能推荐系统正在规划中，
拟通过兴趣画像 + 向量相似度计算实现个性化藏品推荐，进一步展示AI算法能力。

（5）华为云原生融合技术路线（规划中）：项目架构规划向华为云生态迁移，包括
使用华为云ECS部署服务、华为云OBS存储文物影像和3D模型文件、华为云GaussDB
作为数据库服务、华为云ModelArts作为AI模型训练和推理平台，全方位融入华为技术
体系。

三、实施方案
============================================================================

1. 技术可行性分析

技术可行性分析从数据采集、行业知识获取、算力硬件支持和关键技术验证四个维度
展开：

一、数据采集与处理

项目所需数据主要包括：文物基础档案数据（名称、朝代、年代、材质、尺寸、出土
地、描述、文化意义等）、文物影像数据（高清图片、3D扫描模型）、预约业务数据
（用户信息、预约记录）。当前阶段，文物基础数据通过公开的博物馆数据库和标准化
文物资料整理获得，已构建10余件代表性陕西文物藏品数据（涵盖秦兵马俑、唐三彩、
法门寺秘色瓷、汉阳陵陶俑等）。影像数据通过开源图片库和公开3D模型资源构建
原型展示。正式上线阶段，可通过与陕西省文物局合作获取权威数据资源。

二、行业知识获取

AI讲解功能的核心知识来源是藏品档案数据，每件藏品在数据库中存储了详细的描述
文字和文化意义字段。AI模型基于这些结构化数据生成个性化讲解，同时借助DeepSeek
大模型自身在历史、文化、考古领域的预训练知识，实现超出数据库内容的知识补充和
拓展回答。这种方法不需要额外采集外部知识库，即可实现高质量的行业知识讲解。

三、算力与硬件支持

当前阶段，AI推理使用DeepSeek云端API，无需本地GPU算力支持，通过HTTP请求即可
调用大模型能力，极大降低了硬件门槛。服务器端部署于本地开发机（8核CPU + 16GB
内存），可支撑原型系统的并发访问。后续华为云部署阶段，可通过ModelArts平台
按需获取弹性算力（昇腾910/GPU T4等），算力资源灵活可控。

四、关键技术验证

AI智能讲解功能已通过测试验证：使用DeepSeek-chat模型（兼容OpenAI API格式），
通过LangChain4j框架集成，流式输出平均首字延迟<500ms，生成速度约30—50
tokens/s。对话历史通过Redis持久化，支持24小时内多轮上下文。系统在原型测试中
能够对藏品进行准确的识别和讲解，回答内容的相关性和准确性达到可用水平。3D模型
展示功能已集成Google Model Viewer，在主流移动端和桌面端浏览器中均能正常工作。


2. 技术细节

一、系统总体架构

系统采用前后端分离的B/S架构：前端分为用户端（Vue 3 + Vant 4移动端组件库）
和管理端（Vue 3 + Element Plus桌面端组件库），通过HTTP/RESTful API与后端
通信。后端使用Spring Boot 3.2.5框架，数据层采用Spring Data JPA + MySQL 8.0，
缓存层使用Redis，AI层使用LangChain4j集成DeepSeek大模型。认证采用JWT Token
机制，支持用户端和管理端双通道认证。

具体架构层次如下：

    展示层：Vue 3前端应用（用户端H5 + 管理端Web）
    接入层：Nginx反向代理 + 负载均衡
    业务层：Spring Boot RESTful API
    AI层：LangChain4j + DeepSeek大模型 API
    数据层：MySQL（关系数据）+ Redis（缓存 + 会话）

二、AI智能讲解核心技术

（1）RAG式提示词构建：系统根据用户当前查看的藏品，从MySQL数据库动态读取藏品
档案（名称、朝代、年代、材质、尺寸、出土地、描述、文化意义共8个字段），将其
格式化为结构化的系统提示词，注入AI对话上下文。这种方法实现了知识库的外挂式
管理，藏品数据更新后AI回答自动同步更新，无需重新训练模型。

（2）流式对话引擎：使用LangChain4j的StreamingChatLanguageModel接口实现SSE
流式输出，前端通过Fetch API的ReadableStream读取数据流，实现打字机效果的逐字
展示。服务端配置独立线程池（CachedThreadPool）处理AI请求，超时时间设置为
5分钟，确保长对话不被中断。

（3）对话记忆管理：通过实现LangChain4j的ChatMemoryStore接口，将对话历史
序列化为JSON字符串存入Redis，设置24小时自动过期。每轮对话最多保留20条消息
（1条系统提示 + 19轮用户/AI交互），超出时自动裁剪早期消息，保证上下文窗口的
有效性。

（4）图文讲解自动生成：调用AI模型一次性生成结构化的讲解内容（包含标题、摘要、
历史背景、关键知识点、趣味冷知识、文化价值和引导问题），以JSON格式返回前端
渲染。生成失败时自动降级为基于模板的基础讲解，保证服务可用性。

三、3D模型展示技术

集成Google Model Viewer Web Component（@google/model-viewer），支持在浏览器
中直接渲染.glb格式的3D模型，无需安装任何插件。功能包括：自动旋转展示、多点
触控/鼠标拖拽旋转、滚轮缩放、AR模式（支持WebXR的设备）。设置30秒超时保护，
加载失败提供重试按钮和友好的错误提示。

3. 计划和分工

本项目的参赛周期为2026年5月至2026年8月，整体计划和团队分工如下：

一、项目进度计划

    第一阶段：基础功能完善（2026年5月—6月中旬）
    - 藏品CRUD管理、分类管理、预约管理全流程功能
    - 用户端H5（藏品浏览、详情、预约、个人中心）
    - 管理端Web（藏品管理、预约审核、数据统计）
    - 手机验证码登录、JWT认证授权机制

    第二阶段：AI功能开发（2026年6月中旬—7月中旬）
    - LangChain4j + DeepSeek SSE流式AI对话
    - 基于RAG的藏品智能问答
    - AI图文讲解自动生成
    - Google Model Viewer 3D藏品展示
    - 语音播报功能

    第三阶段：华为云集成与优化（2026年7月中旬—8月初）
    - 华为云ECS部署后端服务
    - 华为云OBS存储静态资源（图片、3D模型）
    - ModelArts AI平台集成探索
    - 系统性能优化与压力测试

    第四阶段：文档与演示准备（2026年8月初—8月中旬）
    - 项目文档撰写
    - 演示视频制作（3—5分钟）
    - 线上演示环境搭建
    - 答辩PPT准备

二、团队分工

    成员A（后端开发工程师）：Spring Boot架构设计、数据库建模、RESTful API
    开发、JWT认证、AI服务集成（LangChain4j + DeepSeek）、系统部署运维。

    成员B（前端开发工程师）：用户端H5开发（Vue 3 + Vant 4），负责首页/藏品

    成员E（UI/UX设计与文档）：系统界面设计、用户体验优化、项目文档撰写、
    演示视频制作和答辩材料准备。
